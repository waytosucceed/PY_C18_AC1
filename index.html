Scope of Variables

SCOPE OF VARIABLES

Have you ever wondered why some variables work perfectly inside a function but give you errors when you try to access them outside?

This happens because of something called the scope of variables.

Understanding scope is essential for controlling where your variables can be accessed and used.

A variable's scope refers to the region where we can access the variables.

Based on the scope, variables can be classified into 3 types:

Local Scope
Global Scope
Nonlocal Scope

Local Scope
Variables created inside a function or a block are called local variables. They can only be accessed inside the function.

Let’s have a look at an example:





When we try to print the msg variable inside the show_msg() function, it works, but when we print it outside the function, it gives an error.

Local variables are useful when you want temporary storage that doesn’t affect the rest of your program.

When you need a variable to be accessible across different parts of your program, you can use global variables.

Global Scope
A variable that is not created inside any block but rather in the main body of the code is called a global variable.

Here msg1 is a global variable and msg2 is a local variable.

Global variables can be accessed anywhere in the code. You can understand pretty much why it has the name Global.

When the global and local variable’s name is the same
When a variable with the same name exists in both global and local scopes:

  • Inside the function, Python prioritizes the local variable.
  • Outside the function, the global variable is used.

A nested function is simply a function defined inside another function.

Nonlocal Scope
The nonlocal scope in Python refers to the scope that is neither in the local scope (within the current function) nor in the global scope but is instead in an enclosing function's scope.

For example:

The nonlocal keyword is used in func2() to refer to the x variable in the enclosing function (func1()).

Without nonlocal, func2() would treat x as a new local variable.

When func2() updates x to "Bye", it directly modifies the x variable in the scope of func1().

The func1() function returns the updated value of x, which is "Bye".

Nonlocal scope is useful when working with nested functions and you need to modify a variable in the enclosing function without making it global.

Advanced Functions
🔍 Advanced Functions

● map()
The map() function applies a given function to all the items in an iterable and returns a map object.

You can convert the map object to a list or a tuple using list() or tuple() functions respectively.

This is useful in transforming the elements of an iterable without the help of loops.

Let’s see how we can use the map() function with tuples and lists.

Tuples




The map() function applies the lambda function lambda y: y + 10 to each element in the tuple tup, adding 10 to each element.

The result is a map object, which can be converted into a tuple to see transformed values.
List



The map() function applies the lambda function lambda x: x.lower() to each element in new_list, converting each string to lowercase.

The result is a map object, which can be converted into a list to view the transformed values.

● filter()
The filter() function in Python is used to filter elements from an iterable (like a list, tuple, or set) based on a specified condition.

It returns a filter object which can be converted to a list or tuple using list() or tuple() functions respectively.

It is often used with lambda functions for concise filtering logic.

Let’s see how we can use the filter() function with tuples and lists.

Tuples


The filter() function applies the lambda function, lambda x: x > 20 to each element in tup, returning only the elements greater than 20.

The result is a filter object, which can be converted into a tuple to view the filtered values.
List


The filter() function applies the lambda function, lambda item: 'n' in item to each element in new_list, returning only the elements that contain the letter 'n'.

The result is a filter object, which can be converted into a list to view the filtered values.

Until now, we’ve seen how map() and filter() are useful for applying functions, especially lambda functions, to iterables.

These functions allow us to transform or filter elements based on a condition or operation.

However, zip() is a bit different. Unlike map() and filter(), which require a function (like lambda) to operate on the elements, zip() doesn’t need any extra functions.
● zip()
The zip() function in Python takes multiple iterables and pairs their elements based on their positions.

It returns a zip object of tuples, where each tuple contains one element from each iterable.

This zip object can be converted into a list or tuple using the list() or tuple() functions, respectively.
